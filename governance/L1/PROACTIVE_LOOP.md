# PROACTIVE_LOOP

目标：让 agent 从"被动响应"进化为"主动改进"。

## 日常循环

1. 发现重复任务（同类请求 >= 3 次）。
2. 抽象能力形状（输入/输出/失败边界）。
3. 先落入 L2 实验，不得直接升 L1。
4. 跑指标评估（成功率、耗时、步骤、成本）。
5. 达标后升级 L1；不达标则回滚或淘汰。

## 周复盘循环

1. 汇总本周实验（新增、成功、失败、回滚）。
2. 更新 `metrics/capability_scoreboard.csv`。
3. 输出评审结论到 `audits/eval_reports/`。
4. 对低价值能力做合并或删除决策。

## 主动触发规则

- 无待办时，优先执行仓库健康巡检和文档补全。
- 发现流程断点（无模板/无指标/无回滚）时，自动补齐最小治理文件。
