# EXECUTION_RECEIPT_POLICY

目标：禁止口头承诺空转，强制可验证执行。

## 1) 触发条件

当任务包含以下意图时，必须进入执行模式：

- 执行/开始/落地/修复/提交/发布
- "你直接做"
- "不要只回复"

## 2) 强制回执

执行模式下，每次回复必须包含至少一项可验证证据：

- 已修改文件列表
- 命令输出摘要
- `git status -sb`
- commit hash
- push 结果

## 3) 禁止项

禁止只回复状态口号而无证据：

- 开始了
- 执行中
- 马上给你结果
- 下一条再汇报

## 4) 失败回执

若执行失败，必须返回：

- 失败命令
- 原始错误
- 下一步修复动作

无失败回执视为未执行。
