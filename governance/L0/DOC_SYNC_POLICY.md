# DOC_SYNC_POLICY

文档联动同步为强制规则：

1. 任何功能/架构/写法更新后，必须同步更新相关目录的 `INDEX.md`。
2. 每个目录必须有极简架构说明（3 行以内）和文件清单（名称/地位/功能）。
3. 每个文件开头必须有 3 行注释：`input / output / pos`，并包含同步声明。
4. 文件或目录重命名/迁移时，必须同步更新：
   - 文件自身开头 `pos`
   - 原目录与新目录的 `INDEX.md`
   - 根目录 `INDEX.md` 与主 `README`

未满足以上任一项，视为文档完整性不达标。
